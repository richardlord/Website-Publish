ASDoc for MXML components

Posted on 27 January 2009

I spent a little while yesterday looking for a solution to problems with documenting MXML components with ASDoc in Flex Builder 3. What I found was a number of links that said functions defined in mx:Script tags can be documented, but nothing else. However, after a little thought, I found a way to also document the class itself. Hopefully this will be useful to others in the same situation.

Properties and methods defined in an mx:Script element can be documented in the usual way - by placing the ASDoc comment before the property or method definition.

The class itself can be documented by placing the ASDoc comment inside an mx:Metadata element. The mx:Metadata element's contents are inserted in the class definition immediately before the the class declaration, so a comment here becomes a class-level comment. You'll want to use a CDATA section inside the mx:Metadata element, just as is usual with an mx:Script element.

Properties declared in MXML code can't be documented.

Finally, if you want to read up on the ASDoc improvements coming in Flex 4, the documents are here.
